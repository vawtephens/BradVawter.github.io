User may enter up to 20 different JC's and each JC may have up to 10 unique search terms. During paretoing the program searches the data for the search terms and if the search term is found then the work order is recorded within the JC Bucket. The JC's remain as a pareto definition every time new data is pulled in. Over time JC's can be refined to improve the effectiveness of the data tool. Each search term is configurable so the program will search for the term in the maintenance work order entries for Machine Description, Work Order Descprition, and/or the maintenance work order confirmation.

The user enters the machine names and machine sub systems into three (3) different Machine Category (MC) levels. For instance the three levels for a Car could be Level 1-> Honda Accord, Level 2-> Engine, Level 3-> Spark Plug. Or for instance on a production machine Level 1-> 'Labeller-01' Level 2-> 'Label Drive Assembly', Level 3-> 'Label Drive Motor'. Each of the machine terms can search any of the three maintenance work order entries listed above in JC description. During paretoing the program searches the data for the MC's and if the search term is found then the work order is recorded within the JC Bucket.
